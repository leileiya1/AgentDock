# AgentFlow 接口契约与验收清单 v1.1（Codex 与 Claude 共同遵守）

> 本文档是前后端之间、以及"实现 vs 验收"之间的**唯一权威**。
> 修改流程：任何一方要改契约 → 先在本文末尾 CHANGELOG 登记（日期、发起方、条目、理由）→ 改本文正文 → 再改代码与生成物。未登记的契约偏离在验收中一律记 ❌。

## 4. Tauri Events 契约（Rust → UI）

| 事件名 | payload | 频率约束 |
|---|---|---|
| `task:changed` | `{ task: TaskSummary, from: TaskStatus, to: TaskStatus }` | 每次迁移一条 |
| `run:started` | `RunSummary` | |
| `run:log` | `{ runId: string, batch: AgentEvent[] }` | ≤10 批/秒/run，单批 ≤64KB，已脱敏 |
| `run:finished` | `RunSummary` | |
| `env:changed` | `EnvReport` | |
| `app:error` | `{ scope: string, code: ErrorCode, message: string }` | 后台错误专用 |

`AgentEvent`（适配器归一化产物，亦是 agent-events.jsonl 行格式）：

```ts
  kind: "system" | "assistant_text" | "tool_use" | "tool_result" | "result" | "raw";
  summary: string;            // 单行摘要，UI 列表行
  text?: string;              // 可折叠全文
}
```

## 6. 文件与格式契约

### 6.1 `<repo>/.agentflow/project.toml`（人工维护，可提交）

```toml
schema_version = 1

[[validate.steps]]
name = "test"
argv = ["bun", "test"]          # 必须为 argv 数组，禁止 shell 字符串
timeout_secs = 600

[review]
exclude_globs = ["*.lock", "package-lock.json", "pnpm-lock.yaml", "bun.lockb", "dist/**", "*.min.*"]
max_patch_bytes = 262144

[agents]
extra_allowed_commands = ["bun test", "bun run lint"]   # 转为 Bash(bun test:*) 等白名单项
```

### 6.2 `.agentflow-in/input.md`（Orchestrator 生成，开发 run 输入）

必含区块（模板固定，Codex 实现为模板文件而非字符串拼接）：
`# 任务 TASK-<seq> r<N>` / `## 需求`（title+description+累计人工 guidance）/ `## 项目规则`（内联 rules/*.md）/ `## 历史`（仅返工轮，=history digest）/ `## 边界`（只允许修改本 worktree；禁止改动 `.agentflow/`、`CLAUDE.md`、`AGENTS.md`、CI 配置，除非需求明确要求；不要执行白名单外命令）/ `## 交付`（完成后将结果先写入 `.agentflow-out/result.tmp.json` 再重命名为 `result.json`，schema 如下 + 内联 schema）。

### 6.3 `.agentflow-in/review-input.md`（审查 run 输入）

必含：`## 原始需求`（不含开发者自述）/ `## 审查对象`（commit SHA、base、numstat 汇总、flagged 文件警示段）/ `## diff`（或超限降级说明）/ `## 测试报告` / `## 输出`（review JSON schema 内联 + "最终消息只输出 JSON"指令）。

### 6.4 `result.json` Schema（`packages/schemas/generated/result.schema.json`）

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["schema_version", "task_id", "revision", "status", "summary"],
  "additionalProperties": false,
  "properties": {
    "schema_version": { "const": 1 },
    "task_id": { "type": "string" },
    "revision": { "type": "integer", "minimum": 1 },
    "status": { "enum": ["completed", "failed", "needs_clarification"] },
    "summary": { "type": "string", "minLength": 1, "maxLength": 4000 },
    "question": { "type": "string" },
    "changed_files": { "type": "array", "items": { "type": "string" } },
    "notes": { "type": "string" }
  }
}
```

（`changed_files` 仅信息性；权威以 git diff 为准。`status=needs_clarification` 时 `question` 必填——由 Orchestrator 附加校验。）

### 6.5 `review.json` Schema（`review.schema.json`，同时用于 `codex exec --output-schema`）

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["schema_version", "task_id", "revision", "commit_sha", "decision", "summary", "issues"],
  "additionalProperties": false,
  "properties": {
    "schema_version": { "const": 1 },
    "task_id": { "type": "string" },
    "revision": { "type": "integer", "minimum": 1 },
    "commit_sha": { "type": "string", "minLength": 7 },
    "decision": { "enum": ["pass", "request_changes", "block"] },
    "summary": { "type": "string", "maxLength": 4000 },
    "issues": {
      "type": "array",
      "items": {
        "type": "object",
        "required": ["severity", "title"],
        "additionalProperties": false,
        "properties": {
          "severity": { "enum": ["critical", "high", "medium", "low"] },
          "file": { "type": "string" },
          "line_start": { "type": "integer", "minimum": 1 },
          "line_end": { "type": "integer", "minimum": 1 },
          "title": { "type": "string", "minLength": 1, "maxLength": 200 },
          "description": { "type": "string", "maxLength": 4000 },
          "suggested_action": { "type": "string", "maxLength": 2000 }
        }
      }
    }
  }
}
```

附加校验（Orchestrator）：`decision != "pass"` 时 `issues` 非空；`commit_sha` 必须等于本轮 revision commit，防审查错对象。

### 6.6 `r<N>-tests.json`

`{ "schema_version": 1, "passed": bool, "steps": [{ "name", "argv": [..], "exit_code", "duration_ms", "stdout_tail", "stderr_tail" }] }`

## 7. 命名约定

- 分支：`agentflow/TASK-<seq>`；revision commit message：`[agentflow] TASK-<seq> r<N>: <title>`；merge commit：`[agentflow] merge TASK-<seq>: <title>`。
- commit author：`AgentFlow (<agent>) <agent@agentflow.local>`。
- run_id / 各表 id：UUIDv7 字符串；run 目录名 = run_id。
- worktree 路径：`<worktree_root>/p<projectSeq>/t<taskSeq>`。

## 8. 验收清单（审查对比实现用）
